# Superstack — Full-Stack Development Skill

Build production-ready web products that look like they belong on Awwwards, perform like enterprise software, are hardened against real-world attacks, and convert like they were written by a world-class copywriter.

## Philosophy

Every project should feel like it was crafted by a boutique agency, not generated by AI. This means: editorial layouts, intentional whitespace, bold typography pairings, custom or hand-picked icon sets, micro-interactions that delight, noise/grain textures for depth, and scroll-driven storytelling with Lenis smooth scroll.

### AI Anti-Patterns — avoid these because they instantly signal "AI-generated"

| Anti-Pattern | Do This Instead |
|---|---|
| Symmetric 3-column card grid | Bento grid, asymmetric splits, staggered layouts |
| Uniform spacing everywhere | Intentional rhythm: tight grouping + generous breathing room |
| Generic gradient backgrounds | Noise textures, grain overlays, OKLCH chromatic layering |
| Same animation timing (0.3s ease) | Staggered entrances, spring physics, varied durations |
| Standard Lucide/Heroicons | Phosphor, Tabler, Iconoir, or custom SVGs |
| Stock hero: centered headline + subtitle + button | Asymmetric layouts, split screens, editorial compositions |
| Default shadcn/ui without customization | Heavy customization: colors, radii, shadows, animations |
| Pure white/black backgrounds | Off-white with warm tint, warm dark (slight hue) |
| No scroll storytelling | Lenis smooth scroll + GSAP ScrollTrigger sections |
| Missing textures | Noise/grain overlay, glassmorphism with texture |

## Phase 0: Deep Discovery & Concept Document

Before writing code, gather every requirement. Present questions in logical groups, wait for all answers, then write a `PROJECT-CONCEPT.md` (see `references/project-concept.md` for template).

### Question Groups

**Product & Audience**: What is the product? Who is the target audience? Competitor or inspiration sites?

**Technical Foundation**: Framework, backend, database, auth, state management, extras (file uploads, i18n, dark mode, PWA). See `references/auth-patterns.md` and `references/devops.md`.

**Tracking & Marketing**: Tracking pixels, analytics, server-side tracking, UTM. See `references/tracking-pixels.md`.

**E-Mail System**: Transactional (Resend recommended), marketing, templates needed, automation flows. See `references/email-system.md`.

**Payment** (if applicable): Provider (Stripe recommended), model, trial/coupons, tax handling. See `references/payment.md`.

**Hosting & Deployment**: Platform, environments, preview deploys, domain/SSL. See `references/ci-cd.md`.

**Git & Dev Workflow**: Branch strategy, commit convention, changelog, linting. See `references/git-workflow.md`.

**Platform Features**: Admin panel, roles, content management, API access, notifications. For SaaS: onboarding, billing, multi-tenancy, audit log. See `references/saas-features.md`.

**Compliance & Legal**: GDPR depth, cookie consent (Klaro/Cookiebot), Impressum (DDG since May 2024), WCAG 2.1 AA (EAA since June 2025). See `references/legal-gdpr.md`.

**Design Direction**: Visual mood, brand colors/fonts, animation level, icon style, reference URLs.

After gathering all answers: write `PROJECT-CONCEPT.md` with all decisions and feature matrix (MoSCoW). Get explicit confirmation before Phase 1.

## Phase 1: Copywriting Research & Strategy

Great products are built on great messaging. Before any UI work, nail the copy.

1. **Web research**: Search for competitors, target audience language, industry positioning.
2. **Pain point mapping**: Map 3-5 core pain points — these become the backbone of all copy.
3. **Messaging hierarchy**: Hero headline (transformation, not features), subheadline, section headlines, CTAs.

See `references/copywriting-framework.md` for the full deep-dive.

---

## Phase 2: Design System & Visual Direction

### Core design principles

1. **Does not look AI-generated.** Check the anti-patterns table above. Every project needs at least one creative layout element (bento grid, asymmetric split, horizontal scroll section).

2. **Lenis smooth scroll is standard.** Install and configure for every project. Sync with GSAP ScrollTrigger — this is what separates premium from amateur.

3. **OKLCH color system.** Derive entire palette from a single brand hue by shifting lightness. See `references/design-tokens.md` for the three-layer token architecture and `@theme` setup.

4. **Custom or curated icons.** Phosphor, Tabler, Iconoir, or custom SVGs.

5. **Noise/grain texture.** Subtle SVG noise overlay on hero sections and cards — adds depth and breaks the "too clean" look.

6. **Modern CSS.** Container queries, `:has()`, scroll-driven animations, `@property`.

7. **Responsive from ground up.** Every component works at 375px, 768px, 1024px, 1440px. Use `clamp()` for fluid typography.

8. **Tailwind v4 syntax.** CSS-first config with `@theme`, `bg-linear-to-*`.

See `references/design-patterns.md` for complete patterns, component recipes, and animation techniques.

## Phase 3: Full Scaffold Generation

Generate a complete project the user can `bun install && bun dev` immediately. Use `references/scaffolding-templates.md` to pick the right template for the project type.

### Every scaffold includes

1. Working front-end with real copy (not lorem ipsum), all pages and components
2. Lenis smooth scroll configured with GSAP ScrollTrigger sync
3. Fully responsive at all breakpoints (mobile-first)
4. Every UI state handled — loading (skeletons), error (boundaries + retry), empty (illustration + CTA), custom 404/500
5. Animations wired up (Framer Motion + GSAP), respecting `prefers-reduced-motion`
6. Custom icons — curated or custom SVGs
7. Noise/grain texture on hero and feature sections
8. SEO basics — meta tags, OG images, JSON-LD, semantic HTML, sitemap, robots.txt, canonical URLs
9. Dark mode (if selected) with OKLCH color system and persistence
10. Back-end (if applicable): API routes, DB schema + migrations + seeds, auth flow, Zod validation, rate limiting
11. Multi-environment config: `.env.example`, `.env.development`, `.env.production`
12. README.md with badges, quick start, architecture diagram, env vars table
13. CHANGELOG.md initialized
14. Biome configured for linting + formatting

## Phase 4: IT Security Audit

After scaffold generation, run a full sweep. Check for: injection vulnerabilities (XSS, SQL, NoSQL), auth flaws, authorization gaps (IDOR), data exposure, CSRF, insecure headers, dependency vulnerabilities, environment leaks, API key exposure, webhook signature verification.

Goal: zero Critical and zero High vulnerabilities before delivery. See `references/security-checklist.md`.

---

## Phase 5: Accessibility (WCAG 2.1 AA)

Legally required in the EU since June 2025 (EAA). Key areas: semantic HTML, keyboard navigation with visible focus, ARIA attributes, color contrast (4.5:1 minimum), reduced motion support, form accessibility, 44x44px touch targets. See `references/accessibility.md`.

---

## Phase 6: Performance & Core Web Vitals

**Targets**: LCP < 2.5s, CLS < 0.1, INP < 200ms, FCP < 1.8s, TTFB < 800ms.

Optimize images (next/image, WebP/AVIF), fonts (self-host, subset), JavaScript (code-split, < 100KB gzipped main bundle), CSS (purge unused), server (SSG/ISR, Cache-Control, CDN), animations (GPU-accelerated transforms/opacity only).

---

## Phase 7: Continuous Testing

Tests are written during development, not after. **Unit** (Vitest + Testing Library), **E2E** (Playwright), **API** tests, **MSW** for mocking. See `references/testing-patterns.md`.

## Smart Defaults

When the user has no strong preference, use these without asking:

| Decision | Default | Why |
|---|---|---|
| Framework | Next.js 15 (App Router) | Best ecosystem, RSC, Vercel deploy |
| CSS | Tailwind v4 + shadcn/ui | Fastest DX, customizable |
| Auth | Better Auth | Most complete (Auth.js team joined Sept 2025) |
| Database | Supabase (PostgreSQL) | RLS, realtime, storage built-in |
| ORM | Drizzle ORM | Edge-compatible, type-safe |
| State | Zustand + TanStack Query | Server/client state separation |
| Email | Resend + React Email | Best developer experience |
| Linting | Biome | Replaces ESLint + Prettier, faster |
| Package Manager | Bun | Fastest, built-in test runner |
| Icons | Phosphor Icons | Consistent, 6 weights |
| Analytics | PostHog | Privacy-friendly, feature flags |
| Monitoring | Sentry | Industry standard, tunnel route |
| Hosting | Vercel | Zero-config Next.js deploy |
| Animations | Framer Motion + GSAP | FM for UI, GSAP for scroll |
| Forms | React Hook Form + Zod | Best validation DX |
| URL State | nuqs | Type-safe search params |

Only ask when the choice meaningfully impacts the project direction.

## Interaction style

- Be opinionated — recommend the best approach, explain why, let the user override
- Show your work — explain design decisions
- After Phase 0 confirmation: work autonomously through all phases
- Use parallel agents in every phase
- Output checkpoint summaries after each phase
- Enforce quality gates between phases
- Use conventional commit messages
- When writing German text (copy, legal pages, UI labels, communication): use proper German characters (ä, ö, ü, ß, Ä, Ö, Ü). Never replace them with ae, oe, ue, ss. This applies to all generated content — HTML, JSX, markdown, JSON, database seeds, email templates. Code variables and file names stay ASCII (English).
