# Superstack

A Claude Code skill for building production-ready, full-stack web products with award-winning design, expert copywriting, IT security, testing, CI/CD, accessibility, GDPR compliance, and auto-deployment.

## What it does

Superstack turns Claude into a senior full-stack developer, creative director, security engineer, and DevOps specialist — all in one. It enforces a structured 16-phase workflow with parallel agent execution, quality gates, and smart defaults that takes a project from idea to deployed product without cutting corners.

### Philosophy

Every project built with Superstack should feel like it was crafted by a boutique agency, not generated by AI. No generic hero sections, no "Welcome to our platform" headlines, no cookie-cutter card grids. Instead: Lenis smooth scroll, editorial layouts, intentional whitespace, bold typography, OKLCH color system, noise/grain textures, custom icons, micro-interactions, and scroll-driven storytelling.

## The 16 Phases

| Phase | Name | What happens |
|:---:|---|---|
| **0** | **Deep Discovery & Concept Document** | Mandatory. Gathers ALL requirements across 10 groups. Parallel research agents analyze competitors, design inspiration, tech validation, pricing, and legal requirements. Outputs `PROJECT-CONCEPT.md` with MoSCoW feature matrix. |
| **1** | **Copywriting Research** | Researches competitors and audience, maps 3-5 core pain points, crafts conversion-focused copy using PAS framework, headline formulas, and contextual CTAs. |
| **2** | **Design System** | Design token system (tokens.css), Lenis smooth scroll, OKLCH color system, noise/grain textures, Framer Motion + GSAP animations, custom icons, Tailwind v4, modern CSS. |
| **3** | **Full Scaffold** | Complete production-ready project with real copy, Lenis, responsive at all breakpoints, every UI state handled, animations wired up, noise textures, SEO, dark mode, back-end, Biome linting, README, CHANGELOG. Quality gate: `bun run build` must pass. |
| **4** | **IT Security Audit** | XSS, CSRF, IDOR, SQL injection, auth hardening, API security, webhook verification, dependency audit. Quality gate: zero Critical/High vulnerabilities. |
| **5** | **Accessibility (WCAG 2.1 AA)** | Legally required in EU since June 2025 (EAA). Semantic HTML, keyboard navigation, ARIA, color contrast, reduced motion, 44px touch targets. |
| **6** | **Performance** | LCP < 2.5s, CLS < 0.1, INP < 200ms. Image optimization, font subsetting, code splitting, edge functions. Quality gate: Lighthouse > 90. |
| **7** | **Continuous Testing** | Vitest + Testing Library, Playwright E2E, MSW for API mocking. Tests written alongside features. Quality gate: all tests pass. |
| **8** | **CI/CD Pipeline** | GitHub Actions: lint, type-check, test, build. Preview deploys, environment-specific deployment, Docker builds. |
| **9** | **Git Workflow & Documentation** | Conventional Commits, automatic changelog, README maintenance, API docs, branch protection. |
| **10** | **Tracking & Marketing** | Cookie consent, Google Consent Mode v2, GTM, Meta Pixel + CAPI with event deduplication, TikTok/LinkedIn pixels, privacy-friendly analytics. |
| **11** | **Analytics & Monitoring** | Sentry with tunnel route, Pino structured logging, health check endpoints, uptime monitoring. |
| **12** | **SEO + GEO** | Technical SEO, schema markup, GEO for AI search engines (ChatGPT, Perplexity, Google AI Overviews). |
| **13** | **Deployment** | Auto-deploy to Vercel, Dokploy, Docker. Multi-environment (DEV/STAGING/PROD). Quality gate: site live + SSL. |
| **14** | **Optional Extras** | Cost estimation, DSGVO-quality legal texts, extended documentation (API docs, Storybook, ADRs). |
| **15** | **Post-Launch** | A/B testing setup, heatmap integration, feedback widget, uptime monitoring, performance monitoring, content update guide. |

## Key Features

### Process Excellence
- **Parallel Agent Execution** — Every phase uses multiple agents working simultaneously for maximum speed.
- **Quality Gates** — Automated checks between phases (build, typecheck, test, Lighthouse) — no skipping.
- **Phase 0 Research Sprint** — 5 parallel agents research competitors, design, tech, pricing, and legal before writing the concept.
- **Checkpoint Summaries** — Status report after every phase with decisions, issues, and next steps.
- **Smart Defaults** — Opinionated defaults (Next.js 15, Drizzle, Better Auth, Biome, Bun) when user has no preference.
- **Scaffolding Templates** — Pre-defined starting points for SaaS, Landing Page, E-Commerce, Dashboard, Portfolio.
- **Design Token System** — Every project starts with `tokens.css` deriving all colors from a single OKLCH brand hue.
- **Skill Self-Test** — Comprehensive checklist before delivery (build, design, security, performance, deployment).
- **Lessons Learned Log** — Post-project knowledge capture for continuous improvement.

### Design Quality
- **Lenis smooth scroll** — Standard on every project. Synced with GSAP ScrollTrigger.
- **OKLCH color system** — Perceptually uniform palettes from a single brand hue.
- **Noise/grain textures** — SVG noise overlays for depth. Breaks the "too clean" AI look.
- **Anti-AI design** — No symmetric grids, generic gradients, or default icons. Bento grids, asymmetric splits, editorial compositions.
- **43 reference files** — Comprehensive patterns for every aspect of web development, each with a Table of Contents for quick navigation.

### Technical
- **Better Auth (2026 standard)** — Auth.js team joined Better Auth in Sept 2025.
- **Stripe Embedded Checkout** — Server Actions pattern with Billing Portal, webhooks, EU VAT.
- **React Email + Resend** — Component-based email templates with SPF/DKIM/DMARC.
- **Google Consent Mode v2** — Mandatory. Consent defaults loaded BEFORE any tracking scripts.
- **Facebook CAPI** — Server-side tracking with event deduplication.
- **DDG-compliant Impressum** — German Digital-Dienste-Gesetz (replaced TMG since May 2024).
- **EAA accessibility** — European Accessibility Act in force since June 2025.
- **Biome** — Replaces ESLint + Prettier. Single tool for linting and formatting.

## Installation

### As a Plugin (recommended — includes slash command)

```bash
claude plugin add /path/to/superstack
```

Or from GitHub:

```bash
claude plugin add https://github.com/arrrda8/superstack
```

### As a Skill only

Copy `skills/superstack/` (with `SKILL.md` + `references/`) to `~/.claude/skills/superstack/` (global) or `your-project/.claude/skills/superstack/` (project-local).

## Usage

### Slash Commands

| Command | Purpose |
|---------|---------|
| `/superstack:build` | Start a new project from scratch (16-phase workflow) |
| `/superstack:review` | Full project review against all quality standards (design, code, copy, security, a11y, performance, SEO, compliance) |
| `/superstack:audit` | Focused audit on security, accessibility, performance, or SEO |
| `/superstack:redesign` | Redesign an existing project with premium design standards |
| `/superstack:add` | Add a feature to an existing project (auth, payments, i18n, etc.) |
| `/superstack:shutdown` | Clean session close: scan, clean, update docs, lessons learned, commit & push |

**Examples:**

```
/superstack:build SaaS dashboard for project management
/superstack:review
/superstack:audit security
/superstack:redesign homepage
/superstack:add stripe payments
/superstack:shutdown feat: complete auth implementation
```

### Automatic Triggering

The skill triggers automatically when you describe a web project:

- *"Build me a landing page for my SaaS product"*
- *"Erstell mir eine Webseite fuer mein Startup"*
- *"I need a customer dashboard with auth, billing, and GDPR compliance"*
- *"Make a full-stack MVP for my startup idea"*

## Tech Stack Flexibility

Superstack adapts to your choices during Phase 0 (with smart defaults when you have no preference):

- **Frameworks**: Next.js (default), Nuxt, SvelteKit, Astro
- **CSS**: Tailwind v4 + shadcn/ui (default), CSS Modules
- **ORM**: Drizzle (default), Prisma, raw SQL
- **Databases**: Supabase/PostgreSQL (default), Firebase, PlanetScale, Convex
- **Auth**: Better Auth (default), Clerk, Supabase Auth
- **Hosting**: Vercel (default), Dokploy, Railway, Hetzner, AWS
- **Payments**: Stripe (default), LemonSqueezy, Paddle
- **Email**: Resend (default), Postmark, SendGrid
- **Analytics**: PostHog (default), Plausible, Umami, GA4
- **State**: Zustand + TanStack Query (default)
- **Forms**: React Hook Form + Zod (default)
- **Linting**: Biome (default), ESLint
- **Package Manager**: Bun (default), pnpm, npm
