# Superstack

A Claude Code plugin for building production-ready, full-stack web products with award-winning design, expert copywriting, IT security, testing, CI/CD, accessibility, and auto-deployment.

## What it does

Superstack turns Claude into a senior full-stack developer, creative director, security engineer, and DevOps specialist — all in one. It enforces a structured 13-phase workflow that takes a project from idea to deployed product without cutting corners.

### Philosophy

Every project built with Superstack should feel like it was crafted by a boutique agency, not generated by AI. No generic hero sections, no "Welcome to our platform" headlines, no cookie-cutter card grids. Instead: editorial layouts, intentional whitespace, bold typography, custom icons, micro-interactions, and scroll-driven storytelling.

## The 13 Phases

| Phase | Name | What happens |
|:---:|---|---|
| **0** | **Requirements Gathering** | Mandatory. Gathers ALL requirements upfront in one session across 6 groups: Product & Audience, Technical Foundation, Hosting & Deployment, SaaS Extras, Compliance & Legal, Design Direction. After this phase, the agent works through everything without interruption. |
| **1** | **Copywriting Research** | Researches competitors and audience, maps 3-5 core pain points, crafts conversion-focused copy using PAS framework (Pain → Agitation → Solution), headline formulas, and contextual CTAs. |
| **2** | **Design System** | Visual direction inspired by Awwwards / Dribbble / Codrops. Framer Motion + GSAP animations. Custom or curated icons (Phosphor, Tabler, Iconoir — no default Lucide/Heroicons). Responsive from the ground up. Dark mode support. |
| **3** | **Full Scaffold** | Complete production-ready project: all pages with real copy, responsive at 4 breakpoints, every UI state handled (loading skeletons, error boundaries, empty states, offline, custom 404/500), auth + social login, DB with migrations + seed data, file uploads, search, transactional email templates, i18n, PWA, multi-environment config. |
| **4** | **IT Security Audit** | Continuous auditing + full sweep: XSS, CSRF, IDOR, SQL injection, auth hardening, HTTP security headers, API security, dependency audit. Goal: zero Critical/High vulnerabilities. |
| **5** | **Accessibility** | WCAG 2.1 AA compliance: semantic HTML, keyboard navigation, focus management & traps, ARIA attributes, color contrast (4.5:1), `prefers-reduced-motion`, screen reader support, 44px touch targets, form accessibility. |
| **6** | **Performance** | Core Web Vitals targets: LCP < 2.5s, CLS < 0.1, INP < 200ms, FCP < 1.8s, TTFB < 800ms. Image optimization (WebP/AVIF), font subsetting, code splitting, CSS purging, caching, edge functions. |
| **7** | **Testing** | Vitest + Testing Library unit tests, Playwright E2E tests, API route tests. All generated with the scaffold, scripts in package.json. |
| **8** | **CI/CD Pipeline** | GitHub Actions: lint → type-check → test → build → E2E → deploy. Separate workflows for DEV/TEST/PROD if multi-environment. |
| **9** | **Analytics & Monitoring** | Privacy-friendly analytics (Plausible/Umami/PostHog), Sentry error monitoring, uptime health checks, structured JSON logging with request ID tracing. |
| **10** | **SEO + GEO** | Technical SEO (sitemap, robots.txt, canonical URLs, schema markup, Open Graph). **GEO** (Generative Engine Optimization): optimizing for AI search engines (ChatGPT, Perplexity, Google AI Overviews) — self-contained quotable sections, FAQ schema, SSR for AI crawlers, entity authority signals. |
| **11** | **Deployment** | If credentials provided in Phase 0: auto-deploys to Vercel, Dokploy, VPS (SSH + Nginx/Caddy + PM2), or Docker. Multi-environment deployment. If no credentials: step-by-step deployment docs. |
| **12** | **Optional Extras** | Asked at the end: Cost Estimation (monthly breakdown by usage tier), DSGVO-quality legal texts (Impressum, Datenschutzerklärung, AGB, Cookie-Richtlinie), Extended Documentation (API docs, component library, ADRs). |

## Key Features

- **All questions upfront** — Phase 0 gathers every requirement across 6 structured groups so the agent works autonomously through all phases without interruption.
- **SaaS-aware** — Proactively asks about: Admin Panel, Affiliate/Referral Program, Onboarding Flow, Notification System, Role & Permission System, Billing & Subscriptions, API/Developer Access, Audit Log, Multi-tenancy, Customer Support, Changelog, Status Page.
- **Anti-AI design** — No generic gradients, symmetric grids, or default icon sets. Custom typography pairings, curated icons, award-winning layout patterns from Awwwards/Dribbble/Codrops.
- **Framer Motion + GSAP** — Staggered reveals, magnetic buttons, page transitions, scroll-driven animations, parallax, horizontal scroll, text reveals, number counters.
- **Fully responsive** — Every component tested at 375px, 768px, 1024px, 1440px. `clamp()` for fluid typography. Creative mobile navigation.
- **Real copywriting** — Web research, competitor analysis, pain-point mapping, PAS framework, conversion-focused messaging. No lorem ipsum.
- **Security-first** — Full vulnerability sweep (XSS, CSRF, IDOR, injection, auth, headers, dependencies) before delivery.
- **Accessible** — WCAG 2.1 AA: keyboard navigation, screen readers, focus management, reduced motion, 44px touch targets.
- **Multi-environment** — DEV/TEST/PROD configs with separate `.env` files, database instances, and deployment targets.
- **Auto-deploy** — Provide hosting credentials and it handles deployment end-to-end (Vercel, Dokploy, VPS, Docker).
- **Email templates** — Transactional emails (welcome, password reset, invoice, notifications) built with React Email or MJML.
- **Dark mode, i18n, PWA** — All asked about during brainstorming, all built into the scaffold when requested.
- **GEO-optimized** — Content structured for AI search engines (ChatGPT, Perplexity), FAQ schema, SSR, AI crawler access.

## Installation

### As a Plugin (recommended — includes slash command)

```bash
claude plugin add /path/to/superstack
```

Or from GitHub:

```bash
claude plugin add https://github.com/arrrda8/superstack
```

### As a Skill only

Copy `skills/superstack/` (with `SKILL.md` + `references/`) to `~/.claude/skills/superstack/` (global) or `your-project/.claude/skills/superstack/` (project-local).

## Usage

### Slash Command

```
/superstack:build
```

Or with a project description:

```
/superstack:build SaaS dashboard for project management with auth, Stripe billing, and GDPR compliance
```

### Automatic Triggering

The skill also triggers automatically when you describe a web project:

- *"Build me a landing page for my SaaS product"*
- *"Create a portfolio website for my architecture firm"*
- *"I need a customer dashboard with auth, billing, and GDPR compliance"*
- *"Make a full-stack MVP for my startup idea"*
- *"Build and deploy a fintech SaaS to my Vercel account"*

## Tech Stack Flexibility

Superstack doesn't lock you into one stack. During Phase 0, it asks about and adapts to your choices:

- **Frameworks**: Next.js, Nuxt, SvelteKit, Astro, and more
- **CSS**: Tailwind, CSS Modules, styled-components
- **Databases**: PostgreSQL, MongoDB, Supabase, PlanetScale
- **Auth**: NextAuth, Clerk, Supabase Auth, custom JWT
- **Hosting**: Vercel, Netlify, Railway, Hetzner VPS, AWS, Dokploy
- **Payments**: Stripe, LemonSqueezy
- **Email**: Resend, SendGrid
- **Analytics**: Plausible, Umami, PostHog, Vercel Analytics
- **State**: Zustand, Jotai, Redux Toolkit, TanStack Query
- **Caching**: Redis/Upstash, TanStack Query, ISR/SSG
